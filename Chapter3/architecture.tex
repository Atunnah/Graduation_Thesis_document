\section{Cascaded Detection-Recognition Architecture}
\label{sec:proposed_cascaded_deep_learning_architecture_overview_3_1}
The Vietnamese OCR system developed in this thesis adopts a cascaded two-stage architecture in which text detection and text recognition are performed by two independent, specialized deep learning models connected through a structured inter-stage interface. Rather than pursuing an end-to-end jointly trained architecture that simultaneously optimizes detection and recognition within a single model, this work deliberately separates the two stages to exploit the complementary strengths of RT-DETR and PARSeq, each of which has been specifically optimized for its respective subtask. This architectural choice reflects both practical and principled considerations that are elaborated throughout this section.

End-to-end OCR architectures, which directly map input document images to output character sequences without explicit intermediate detection, have been explored in works such as CRNN \cite{Shi2015} and more recently in attention-based architectures that process entire document images. While these approaches offer the theoretical advantage of joint optimization, they impose strong assumptions about document layout and text line arrangement that limit their applicability to the diverse Vietnamese document types targeted in this thesis. Furthermore, end-to-end models trained on fixed input resolutions struggle to generalize across documents with varying text densities, font sizes, and orientations, all of which are characteristic of real-world Vietnamese administrative and scene text documents. In contrast, a cascaded design decouples the spatial localization problem from the sequence transcription problem, allowing each component to be independently optimized, fine-tuned, and evaluated. This modularity also facilitates targeted improvement of individual stages without requiring full system retraining, which is particularly valuable given the limited availability of annotated Vietnamese OCR data discussed in Chapter \ref{chapter:related_works}.

As illustrated in Figure \ref{fig:proposed_cascaded}, the overall cascaded architecture proceeds as follows. Given an input document image of arbitrary resolution and layout, the first stage employs a fine-tuned RT-DETR V2 detector \cite{Zhao2024} to localize all text line regions as oriented bounding boxes, each parameterized by center coordinates, width, height, and a rotation angle encoded in sine-cosine space. The second stage passes each cropped and rectified text line image to a fine-tuned PARSeq recognizer \cite{Bautista2022}, which transcribes the visual content into a Unicode character sequence using its permutation-based autoregressive decoder. The final output is a structured collection of recognized text strings associated with their spatial positions in the source document, enabling downstream applications such as information extraction, indexing, and document reconstruction.

\subsection{End-to-End Pipeline Design}
\label{sec:end_to_end_pipeline_design}
Whereas Section~\ref{sec:approach_1_2_1} introduced the cascaded architecture at a overview level, this section details the concrete data transformations, tensor shapes, and implementation contracts at each inter-stage interface. The full processing pipeline from raw input image to final recognized text comprises five sequential stages: image ingestion and preprocessing, oriented text line detection, bounding box post-processing and NMS, perspective-corrected cropping, and character sequence recognition. Each stage is implemented as an independent module with well-defined input and output interfaces, enabling the system to be executed as a unified inference pipeline through a single command-line entry point. As detailed in Figure \ref{fig:proposed_pipeline}, this pipeline systematically transforms intermediate data structures and shapes at each interface, the mechanics of which are elaborated in the following stages.

    \label{fig:proposed_pipeline}
\end{figure}

In the first stage, the input image is loaded and converted to RGB format before being passed to the RT-DETR image processor, which resizes it to a fixed resolution of 800 $\times$ 800 pixels while preserving aspect ratio through padding. This standardized resolution is required by the RT-DETR backbone and ensures consistent feature map dimensions throughout the detection network. The processed image tensor is then forwarded through the fine-tuned RT-DETR V2 model, which produces a set of candidate detections comprising classification logits, normalized bounding box coordinates in center-width-height format, and predicted sine-cosine angle pairs from the auxiliary angle regression head.

In the second stage, candidate detections are filtered by a confidence threshold applied to the sigmoid-activated classification logits, retaining only detections with scores exceeding a configurable minimum. The retained bounding boxes and their associated angles are then subjected to Non-Maximum Suppression (NMS) to eliminate duplicate detections of the same text line. When the model has been trained with the oriented angle head, rotated IoU is efficiently computed within the PyTorch framework using \texttt{torchvision.ops.box\_iou\_rotated} to account for the angular component of box overlap, providing more accurate suppression of overlapping oriented detections than axis-aligned IoU would permit.

In the third stage, the surviving detections are sorted by their vertical position to establish a top-to-bottom reading order consistent with standard document layout conventions. Adjacent detections sharing more than 40 percent of their minimum height in vertical overlap are grouped into the same logical text line, and within each group, detections are further sorted by horizontal position to establish left-to-right reading order. This spatial grouping and ordering step reconstructs the logical document structure from the set of individually detected text regions, enabling the final output to reflect the natural reading sequence of the document.

In the fourth stage, each detected text line region is cropped from the original input image using a perspective warp rather than a simple rectangular crop. Given the oriented bounding box parameters, the four corners of the rotated rectangle are computed and used as the source points of a perspective transformation matrix, which maps the rotated text region to an upright rectangular image of dimensions equal to the detected box width and height. This perspective rectification ensures that the cropped text line image passed to the recognizer is approximately horizontal regardless of the original text orientation, satisfying the assumption of the PARSeq recognizer that input images contain horizontally arranged character sequences.

In the fifth and final stage, each rectified text line image is preprocessed by the PARSeq image transform pipeline, which resizes it to the model's expected input dimensions of 48 $\times$ 1024 pixels while preserving aspect ratio through symmetric padding, normalizes pixel values using ImageNet statistics, and converts the result to a floating-point tensor. The preprocessed tensor is forwarded through the fine-tuned PARSeq model in autoregressive decoding mode, producing a predicted character sequence that is decoded from token indices to Unicode strings using the Vietnamese-extended tokenizer. The recognized strings from all text line regions are concatenated in reading order and written to the output file alongside a visualization image showing the detected oriented bounding boxes and recognized text overlaid on the original document.

\subsection{Inter-stage Data Flow and Cropping Strategy}
\label{sec:inter_stage_data_flow_and_cropping_strategy_3_1_2}

The interface between the detection and recognition stages is the most critical design point in the cascaded architecture, as errors introduced at this boundary propagate irreversibly into the recognition output. The inter-stage data flow must correctly transform the geometric detection outputs of RT-DETR into image crops that satisfy the input assumptions of PARSeq, while minimizing the introduction of background noise, character truncation, and geometric distortion.

The detection stage outputs bounding boxes in normalized center-width-height format, converted to pixel-space coordinates by multiplying by image dimensions. Given a detected text line with center $(c_x, c_y)$, width $w$, height $h$, and rotation angle $\theta$ encoded as $(\sin\theta, \cos\theta)$, the four corners of the oriented rectangle are computed as:

\begin{equation}
\mathbf{p}_i = \begin{pmatrix} c_x \\ c_y \end{pmatrix} + R(\theta) \begin{pmatrix} \pm w/2 \\ \pm h/2 \end{pmatrix}, \quad R(\theta) = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}
\end{equation}

These four corner points serve as the source quadrilateral for a perspective transformation computed using \texttt{cv2.getPerspectiveTransform}, which maps the rotated text region to a canonical upright rectangle of size $w \times h$ pixels. The resulting crop contains the text line content with minimal background inclusion and no angular distortion, providing a clean input for the recognition stage.

The cropped text line image is then resized to the PARSeq input dimensions of $48 \times 1024$ pixels using an aspect-ratio-preserving resize followed by symmetric zero-padding on the shorter dimension. This padding strategy ensures that characters are not stretched or compressed horizontally, which would distort the visual appearance of Vietnamese diacritical marks and reduce recognition accuracy. The padded image is normalized using mean and standard deviation values of $(0.485, 0.456, 0.406)$ and $(0.229, 0.224, 0.225)$ respectively, matching the ImageNet pretraining statistics of the ViT backbone used by PARSeq.

\subsection{Error Propagation Analysis across Stages}
\label{sec:error_propagation_analysis_across_stages}

A fundamental limitation of cascaded architectures relative to jointly trained end-to-end systems is that errors generated in the first stage cannot be corrected by the second stage, as the recognizer operates solely on the cropped image produced by the detector without access to the original detection confidence or bounding box uncertainty. Understanding the failure modes introduced at each stage and their downstream consequences is therefore essential for diagnosing system-level recognition errors and motivating the optimization strategies described in Section \ref{sec:optimization}.

Detection errors manifest in three principal forms. The first is missed detection, in which a text line present in the document is assigned a confidence score below the filtering threshold and is therefore absent from the set of crops passed to the recognizer. Missed detections produce silently dropped text in the final output, which is particularly harmful in administrative document contexts where every field must be transcribed. The second form is spurious detection, in which a non-text region is incorrectly detected as a text line and passed to the recognizer, which will attempt to transcribe visual noise or graphical elements as character sequences. The third and most consequential form for recognition quality is inaccurate localization, in which the detected bounding box or rotation angle deviates from the true text line extent, causing the perspective-corrected crop to include excess background, truncate characters at the box boundary, or introduce residual rotation that the recognizer must implicitly compensate for.

Among these error types, angular localization errors are particularly harmful for Vietnamese text recognition because residual rotation in the cropped image displaces diacritical marks from their expected vertical positions relative to base characters. A rotation error of even a few degrees can cause a dot below or hook above to fall outside the receptive field of the patch embedding that encodes the corresponding base character, effectively making the diacritical mark invisible to the ViT encoder. This motivates the use of oriented bounding box detection with GWD loss in this work, as tighter angular localization directly reduces the residual rotation in inter-stage crops and thereby improves diacritical recognition accuracy in the second stage.

Recognition errors introduced in the second stage are conditioned on the quality of the crop received from the first stage. For well-localized crops, PARSeq's permutation-based decoder and bidirectional linguistic context provide strong robustness against individual character ambiguities. However, for crops with significant background inclusion or character truncation, the recognizer may produce plausible but incorrect transcriptions that are difficult to detect without ground truth reference, as they often constitute valid Vietnamese word forms under the extended vocabulary.
